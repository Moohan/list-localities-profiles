## 2024-05-23 - [Optimization: R Report Pipeline Hoisting]
**Learning:** In large-scale R report generation pipelines (e.g., Public Health Scotland's Locality Profiles), identifying 'global-level', 'partnership-level', and 'locality-level' data operations is critical for performance. Hoisting global data loading (RDS/Parquet) and partnership-level filtering (HSCP-wide lookups/joins) out of the innermost locality loop can reduce redundant I/O and computations by a factor of 10x-100x depending on the number of localities.
**Action:** When refactoring R scripts sourced in loops, decompose them into tiered modules (e.g., `2a. loading.R`, `2b. hscp_manipulation.R`, `2c. locality_table.R`) and use explicit object cleanup (`rm()`) at each level's boundary to prevent memory accumulation while preserving necessary partnership-level artifacts for downstream report generation.

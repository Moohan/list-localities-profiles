## 2025-02-28 - [Hoisting Services map and data manipulation]
**Learning:** In R reporting pipelines using a nested loop structure (HSCP -> Locality), generating static assets like maps in the outer loop provides a significant performance gain. `Services/3. Service HSCP map.R` was taking ~10-20 seconds per partnership due to map tile fetching and rendering. Hoisting this, along with the data manipulation that provides its markers, saves considerable time when rendering profiles for many localities.
**Action:** Always check if a script sourced in a locality-level loop produces an output that is identical for all localities in the same partnership. If so, hoist it to the outer loop and ensure variables are created before the environment preservation checkpoint (`loop_env`).
